How to create your own bots.

I have Used Java and Spring boot to create these bots. Spring boot is not compulsory but it makes it slightly easier.

Telegram is a popular messaging platform known for its security and rich set of features. Telegram Bots allow developers to interact with users on the platform programmatically, providing a seamless and automated experience. In this tutorial, we will walk through the process of creating a Telegram bot using Java and Spring Boot, allowing you to harness the power of both technologies to build a functional and interactive bot.
Link to Official BotFather -> 
Link to Official Documentation ->

Prerequisites
Before we start, make sure you have the following installed on your system:
1. Java Development Kit (JDK) 8 or higher
2. IntelliJ IDEA or any other preferred IDE
3. Maven or Gradle for dependency management
4. Telegram account

Setting Up the Telegram Bot

1.Create a Telegram Bot:
To create a bot on Telegram, you need to interact with the "BotFather" on the Telegram platform. Open Telegram and search for "BotFather" in the search bar. Start a chat with BotFather and use the /newbot command to create a new bot. Follow the instructions provided by BotFather to choose a name and username for your bot. Once created, BotFather will provide you with a unique API token. Save this token as we will use it later to interact with our bot programmatically.

2.Set Up the Spring Boot Project:
Open your preferred IDE and create a new Spring Boot project. You can either use the Spring Initializr (https://start.spring.io/) or create the project from scratch. Make sure to include the required dependencies for Spring Web and Telegram Bots.

3.Add Dependencies:
If you are using Maven, add the following dependencies to your pom.xml file:

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>
<dependency>
    <groupId>org.telegram</groupId>
    <artifactId>telegrambots</artifactId>
    <version>5.2.0</version>
</dependency>



If you are using Gradle, add the following dependencies to your build.gradle file:

implementation 'org.springframework.boot:spring-boot-starter-web'
implementation 'org.telegram:telegrambots:5.2.0'



1. Create the Telegram Bot Class: In your project, create a new Java class for your Telegram bot. For example, name it MyTelegramBot. This class will extend the TelegramLongPollingBot class provided by the Telegram Bots library. Implement the required methods, including getBotUsername(), getBotToken(), and onUpdateReceived(Update update).


import org.telegram.telegrambots.bots.TelegramLongPollingBot;
import org.telegram.telegrambots.meta.api.objects.Update;
import org.telegram.telegrambots.meta.exceptions.TelegramApiException;
public class MyTelegramBot extends TelegramLongPollingBot {
    @Override
    public String getBotUsername() {
        // Return the username you set for your bot during creation
        return "YourBotUsername";
    }
    @Override
    public String getBotToken() {
        // Return the API token provided by BotFather
        return "YourBotToken";
    }
    @Override
    public void onUpdateReceived(Update update) {
        // Handle incoming updates and respond accordingly
        // For example, you can access the user's message using update.getMessage().getText()
        // and send a response using execute(SendMessage message)
        // Don't forget to handle any exceptions that might occur during the process
    }
}



2. Create a Controller: In Spring Boot, create a controller class to handle incoming HTTP requests from Telegram. This controller will process updates and forward them to the MyTelegramBot class.


import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestBody;
import org.springframework.web.bind.annotation.RestController;
import org.telegram.telegrambots.meta.api.objects.Update;
@RestController
public class TelegramBotController {
    private final MyTelegramBot telegramBot;
    public TelegramBotController(MyTelegramBot telegramBot) {
        this.telegramBot = telegramBot;
    }
    @PostMapping("/webhook")
    public void onUpdateReceived(@RequestBody Update update) {
        telegramBot.onUpdateReceived(update);
    }
}


3.Configure the Bot: Open your application.properties file and add the following configuration to set up the Telegram webhook:

# Replace <your-bot-username> with the username you set for your bot during creation
telegram.bot.username=<your-bot-username>
# Replace <your-bot-token> with the API token provided by BotFather
telegram.bot.token=<your-bot-token>



4.Deploy the Bot: Deploy your Spring Boot application to a server or cloud platform. Make sure your server is accessible over HTTPS as Telegram requires webhooks to be served over a secure connection.

5.Set the Webhook URL: Set the webhook URL for your Telegram bot using the following API call or you can use the setWebhook method in the BotFather chat:


curl -F "url=https://your-server-url/webhook" "https://api.telegram.org/bot<your-bot-token>/setWebhook"

Make sure to replace your-server-url with the actual URL of your deployed Spring Boot application.


Conclusion

Congratulations! You have successfully created a Telegram bot using Java and Spring Boot. Your bot is now ready to receive updates from Telegram users and respond accordingly. You can further enhance your bot's functionality by adding more logic to the onUpdateReceived method and implementing various Telegram Bot API features.
Remember to handle exceptions and errors properly, as well as secure sensitive data, such as the bot token, during production deployment. Happy bot building!
